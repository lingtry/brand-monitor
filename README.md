# 🛡️ Brand Monitor — 品牌舆情监控系统

全网多源自动化品牌舆情监控，**每小时**扫描一次，检测到负面信息立即邮件告警。

> 🆓 完全免费 · GitHub Actions 7×24 云端运行 · 电脑关机也监控

## 🔬 负面检测引擎

### 三级分类体系

| 级别 | 关键词示例 | 措施 |
|------|----------|------|
| 🔴 **高危** | 爆炸、起火、召回、诈骗、假货、电死人… | 邮件标题加 🚨 前缀 |
| 🟡 **中危** | 质量问题、投诉、售后差、虚标、充不进电… | ⚠️ 前缀标记 |
| 🟢 **低危** | 不好用、失望、断连、做工差… | 一般关注 |
| ⚪ 中性 | 普通提及、测评、资讯 | 仅供参考 |

### 行业定制关键词

针对 Baseus 产品特性，特别监控：
- 安全类：爆炸、起火、冒烟、漏电、烧手机
- 品控类：充不进电、虚标容量、功率虚标、发烫
- 售后类：拒绝保修、踢皮球、售后无门

## ⚙️ 快速部署

### 第1步：创建 GitHub 仓库

在 GitHub 新建仓库 `brand-monitor`，将本项目推送上去。

### 第2步：配置 Secrets

**Settings → Secrets and variables → Actions → New repository secret**

| Secret | 值 |
|--------|-----|
| `EMAIL_ADDRESS` | QQ邮箱地址 |
| `EMAIL_PASSWORD` | QQ邮箱 SMTP 授权码 |
| `RECIPIENT_EMAIL` | 接收报告的邮箱 |
| `BING_API_KEY` | *可选* Bing Search API Key（[Azure 免费申请](https://portal.azure.com)） |

> Bing Search API 免费层 1000次/月，不配也能跑，只是搜索源少一个。

### 第3步：手动触发测试

Actions → Brand Monitor → Run workflow

## ⏰ 执行频率

北京时间 **8:00 ~ 23:00**，每小时一次，共 16 次/天。

> 需修改频率？编辑 `.github/workflows/brand-monitor.yml` 中的 `cron`。

## 🔧 自定义品牌

编辑 `main.py` 顶部的配置：

```python
BRAND_NAME = "你的品牌英文名"
BRAND_NAME_CN = "品牌中文名"
BRAND_NAMES = [BRAND_NAME, BRAND_NAME_CN, "常见拼错", ...]
BRAND_PRODUCTS = ["产品1", "产品2", ...]
```

搜索查询模板在 `SEARCH_QUERIES`，负面词典在 `NEGATIVE_HIGH/MEDIUM/LOW`。
